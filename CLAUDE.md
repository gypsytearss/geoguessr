# Project Guidelines

## Branching & Git Flow

This project uses **GitHub Flow**:

1. Branch off `main` → `feature/my-feature` (or `fix/my-fix`, `hotfix/my-hotfix`)
2. Do work, commit, push
3. Open PR → `main`
4. Review & merge
5. Delete branch

Never commit directly to `main`.

## Commit Guidelines

- Keep commits small and self-contained — ideally 100–300 lines of code
- Each commit should add one incremental, logical element (not a mix of unrelated changes)
- Prefer many small commits over one large commit per PR
